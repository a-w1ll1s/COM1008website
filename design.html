Introduction

When I was designing this website, I wanted to create a colourful, modern-looking website. To do this, I first decided on a colour palette that would be used throughout. I chose the grey and pink as it had a modern feel while still being interesting and vibrant. Grey also had a nice contrast with the white text which made it more accessible. I then decided on the design of the website elements. I found that giving each element its own visible area split up the website into more readable chunks and giving each area rounded corners made it modern and smooth.

I was able to design the website with Mobile-First Responsive Web Design by using Firefox’s Responsive Design Mode tool, which allows you to set the size of the viewport. This gave me the opportunity to see how the website would look on many different devices and made creating and tweaking the website for a phone much simpler.

Site Map

I chose this site map design as I believe it helps to make the website easier to understand. The different menu systems group the pages into website content (sidebar nav) and website information (footer nav) for logical navigation. This site map also adheres to the 3-click rule, as all content can be reached with fewer than 3 clicks, which means finding information or a specific page is a lot less confusing for users.

Design Mock-ups

My initial designs started with the mobile design as this website was designed as Mobile-First. It is a linear design with a header, elements, and a footer – I decided on the menu systems later. I believe this design is the best for a mobile phone viewport as it doesn’t clutter the screen and each element has plenty of space which helps with readability. The wide viewport design is very similar to the mobile design as the viewport was still limited in size. I decided to put a breakpoint between the wide display and the desktop display so that I could create the design shown in the mock-ups. The elements are no longer linear, but on a desktop viewport it is still legible.

Menu System

I decided to use the off-canvas navigation method from responsivenavigation.net as it allowed me to display more content on the page. This is because it removes the area for the navigation menu as well as the navigation links. It also helps to solidify the modern website theme that I was going for. I got my code from W3schools.

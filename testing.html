Optimisation

My website is quite simple in its design and content which means the pages load at a fast rate. Therefore, not much optimisation is required. The quiz page is an exception because it loads 9 images when opened which could cause delay on slower devices. To optimise this page and its image loading, I used compressed images or WEBP images as they are easier to load than PNGs or JPEGs.

Browser Testing

To test the responsive design of my website, it was important to view it on different devices. However, because my website was not available over the internet, having to download new HTML and CSS every time I wrote new code or tweaked existing code made testing on other physical devices very inefficient. To solve this issue, I used Firefoxâ€™s Responsive Design Mode. This mode allows you to choose the viewport size and the website acts accordingly. For example, if you set the viewport to a mobile phone screen size the website will act as it would on the mobile phone. This was very helpful for designing and testing the website as it streamlined the process of updating code.

Accessibility

To test accessibility on the website, I used the WAVE plug-in:

WAVE, 2022, WAVE Web Accessibility Evaluation Tools, Accessed: 11/2022, { https://wave.webaim.org/}

This screenshot shows the results of the WAVE evaluation of the home page of the website. As shown, this page has no errors, good contrast and 4 accessibility features like alternative text for pictures. However, there are two alerts. One alert applies to the alternative text for the logo. Because the logo is not essential for the understanding of the content, WAVE considers alternative text for it to be unnecessary. I decided to keep the alternative text in as the logo is at the top of the page and the text would not disrupt any important content. The other alert was related to the menu button. WAVE suggested that it may not accessible. However, I used an image for the menu button, so the alternative text solves this issue.
